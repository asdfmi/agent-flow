## Module Overview
- Workflows list and detail page showing saved workflows and entry points to the builder.

## Responsibilities
- Fetch and render workflow summaries.
- Provide navigation into workflow detail or builder views.
- Handle empty states and refresh interactions.

## Interfaces
- Entry: `main.jsx` bootstraps `<WorkflowsPage />`.
- Components under `components/` render tables, filters, and action buttons.

## Depends On
- Hooks like `useWorkflows`, shared components for layout, and workflows API clients.

## Used By
- Operators browsing existing workflows in the portal UI.
- Other pages linking users to workflow listings.
