## Module Overview
- Portal workspace providing the operator-facing API, internal event bridge, and React UI assets.

## Responsibilities
- Persist workflows, steps, runs, and metrics through Prisma.
- Serve REST endpoints and internal webhooks via Express 5.
- Build and ship the React UI bundle under `ui/dist`.

## Interfaces
- `pnpm run dev:api` – start the API server (`server/index.js`).
- `pnpm run dev:ui` – watch/build the Vite UI bundle (`ui/vite.config.js`).
- HTTP: `GET /api/workflows`, `POST /api/workflows/:id/run`, `POST /internal/runs/:runId/events`, `GET /ws` (WebSocket upgrade).

## Depends On
- Prisma Client (`portal/prisma/schema.prisma`) for database access.
- Express middleware, WebSocket server (`ws`), and environment variables `DATABASE_URL`, `INTERNAL_SECRET`, `CRAWLER_BASE_URL`.
- Build tooling: Vite, React 19, MUI.

## Used By
- Root workspace scripts in `package.json`.
- Crawler service (`crawler/index.js`) posting run events back to `/internal`.
- Operator UI consumers loading assets from `portal/ui/dist`.
